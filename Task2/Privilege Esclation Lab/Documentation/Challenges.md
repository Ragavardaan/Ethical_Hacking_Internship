Challenge Description



Lab Scenario



You are a Junior Security Analyst at SecureTech. During a routine security assessment, you have been provided with access to an internal Ubuntu server using a low-privileged user account.



Your objective is to assess the system, identify privilege-related misconfigurations, and understand how improper system administration can expose sensitive resources. Throughout the exercise, you are expected to enumerate the system, inspect users, groups, directories, permissions, and administrative configurations before identifying the intended privilege escalation path.



This lab is designed for educational purposes only and should be performed within the provided virtual machine.



Challenge Objectives



The learner should successfully complete the following tasks:



Challenge 1 – User Enumeration



\- Identify the current logged-in user.

\- List all available users.

\- Identify the groups assigned to the current user.





Challenge 2 – System Enumeration



\- Identify the operating system version.

\- Inspect the directory structure.

\- Locate important system files.

\- Understand the purpose of each department directory.





Challenge 3 – Permission Analysis



\- Examine file ownership.

\- Inspect Linux permissions.

\- Identify which files can be accessed by different users.





Challenge 4 – Sudo Enumeration



\- Check the sudo privileges assigned to the current user.

\- Understand how delegated administrative permissions work.





Challenge 5 – Security Analysis



\- Analyze the overall system configuration.

\- Identify security weaknesses.

\- Recommend security improvements.



Expected Learning Outcome



After completing all the challenges, learners should understand:



\- Linux user management

\- File permissions

\- Directory permissions

\- Sudo configuration

\- Principle of least privilege

\- Importance of proper administrative configuration

\- Basic privilege escalation methodology





Difficulty Level



Easy – Beginner Friendly



This lab is intended for students who have basic Linux knowledge and want to understand privilege escalation concepts through guided practical exercises.
